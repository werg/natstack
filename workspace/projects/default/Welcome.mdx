# Welcome to your default vault

This is the **default Spectrolite vault** — a folder of MDX files backed by a
local git repo. It lives at `projects/default/` in your workspace and is
ready to edit out of the box.

## What you can do

1. **Edit prose** in this rich-text view — formatting renders inline,
   JSX components render live, and `[[wikilinks]]` resolve as you type.
   Use the **diff/source toggle** in the toolbar when you want to drop
   into raw MDX.
2. **@-mention the resident agent** by typing `@` — it stays quiet by
   default and only speaks when it has something to say. Mention it in a
   sentence describing what you want changed, then click **Flush** (or
   pause for 1.5 seconds). The agent reads your diff and edits the file
   in-place.
3. **Link** to other notes with `[[Wikilink Syntax]]`. Click an unresolved
   link to create the note.
4. **Commit** any time from the strip at the bottom. The "Suggest message"
   button asks the agent to draft a commit message from the diff.

## Add another vault

Click the vault name in the top-left header to switch. Create new vaults
from the picker — they live as siblings under `projects/`.

## Stateful components

Components in the doc can persist their own state into the frontmatter
via `useDocState`. The value is read on open and merged back into the
`state:` block ~600 ms after the last change, so the agent (and a
future you opening this file) sees the same value.

```mdx
export const Greeter = () => {
  const [clicks, setClicks] = runtime.useDocState("greetingClicks", 0);
  return (
    <Button onClick={() => setClicks(clicks + 1)}>
      Hello — clicked {clicks} times
    </Button>
  );
};

<Greeter />
```

The button renders live in the editor. The current value lives in this
doc's frontmatter under `state.greetingClicks`.

## Declaring dependencies

Add npm or workspace packages to the YAML frontmatter and they get loaded
into the sandbox automatically:

```yaml
dependencies:
  "date-fns": "npm:^2.30.0"
  lodash: "npm:^4.17.21"
```

Inline JSX in this doc — and the agent's `eval` calls in the context of
this doc — can use those packages without redeclaring imports.
